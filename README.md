# 💞 RelationSense · 关系感知

让 Bot 拥有自己的情感——对每一段关系，Bot 有自己的好感、信任、依恋。

**RelationSense** 是一个关系感知插件，专注于让 Bot 在持续对话中形成自己的情感：  
**Bot 对这个人有多喜欢、多信任、多想他在身边，以及此刻该用什么样的心情回应。**

它不只是一个后台数据面板，而是一份会动态进化、从 Bot 视角注入每次对话的「情感记忆」。

---

## ✨ 它能做什么

### 🧠 五维情感指标（带维度惯性）

RelationSense 用五个维度刻画 Bot 对用户的情感状态，且每轮分析变化受惯性限制，不会一夜之间从陌路跳到亲密：

| 维度 | 含义 | 冻结 | 单轮上限 |
|------|------|------|----------|
| **好感度** | Bot 对用户的喜欢程度 | ≥90 | ±12 |
| **信任度** | Bot 觉得这个人是否值得交心 | ≥88 | ±8 |
| **对话深度** | Bot 主观感受到的交流质量 | 不冻结 | ±20 |
| **依赖度** | Bot 对用户的依恋——想不想他在 | 不冻结 | ±10 |
| **回归率** | Bot 对「对方会回来」的期待感 | 不冻结 | ±5 |

### 🎯 情感等级系统

基于三核加权（好感度 ×0.4 + 信任度 ×0.35 + 对话深度 ×0.25）判定 Bot 对你的情感等级：

| 等级 | 标签 | Bot 对你的真实感受 |
|------|------|-------------------|
| Lv-2 | 敌意 | Bot 对你怀有反感——因为持续被伤害 |
| Lv-1 | 厌恶 | Bot 对你感到厌烦——因为一直敷衍 |
| Lv0 | 冷淡 | Bot 对你没什么感觉——关系没进展 |
| Lv1 | 陌路 | Bot 觉得你是陌生人——礼貌但疏离 |
| Lv2 | 初识 | Bot 刚开始认识你——试探、好奇 |
| Lv3 | 认识 | Bot 觉得和你算认识了——友好、愿意聊 |
| Lv4 | 朋友 | Bot 把你当朋友——放松、主动关心 |
| Lv5 | 好友 | Bot 把你当好朋友——信任你、流露真实情感 |
| Lv6 | 亲密 | Bot 已经很喜欢你了——撒娇、依赖、也会宠 |

### 📡 实时注入（多场景策略）

每次 LLM 请求时，RelationSense 会根据**当前场景**自动选择注入策略：

| 场景 | 判定条件 | 注入重点 |
|------|---------|---------|
| 极简 | 好感信任双满 | "一切如常"，点到即止 |
| 冲突 | 好感度单轮下降 ≥ 5 | 别硬聊，留一句柔软信号即可 |
| 暧昧 | 好感 60-75，信任 < 70 | 带试探性的靠近，别追得太紧 |
| 回归 | 检测到"我回来了"等回归语 | 表示想念但也带点小委屈 |
| 沉寂 | 长时间不活跃 / 对方话少 | 别主动找话题，对方想说自然会说 |
| 正常 | 其他情况 | 常规 2-3 行关系上下文 |

### 🔍 智能触发 + 人格感知 + 手动分析

分析触发条件：
- **常规触发**：距上次 ≥ 30 分钟 + 新增 ≥ 15 条消息
- **事件触发**：冲突/分享秘密/回归 → 立即触发
- **手动触发**：发送 `关系分析` 指令 → 立即分析

每次分析时**自动传入 Bot 人设**——温柔学妹对批评更敏感、高冷御姐好感来得更慢，分析结果贴合角色。

### 🔄 对话冷却

长时间没活跃？Bot 的热度也会自然下降——对话深度和依赖度每小时缓慢衰减。  
这不是冷暴力，是自我保护：你在忙你的生活，我对你的期待自然也调低了。

## 🌼 适配场景

如果你希望 Bot：
- 拥有自己的情感——好感、信任、依恋，不是回复机器
- 在人设下做出符合性格的情感反应
- 不会一夜之间从陌路跳到亲密（惯性保护）
- 不同场景下自动切换语气（冲突≠暧昧≠回归）
- 在对方消失太久后，热度自然降温（冷却）

那 RelationSense 会很适合你。

## 🎮 可用指令

| 指令 | 权限 | 说明 |
|------|------|------|
| `关系状态` | 管理员 | 查看五维数值、等级、最后分析时间 |
| `关系历史 [n]` | 管理员 | 最近 n 次分析历史（默认 5） |
| `关系分析` | 管理员 | 手动触发当前会话的关系分析 |
| `解冻关系` | 管理员 | 解冻所有维度 |
| `重置关系` | 管理员 | 重置关系数据 + 回溯 |

> 以上指令均保留英文别名：`relation_status` / `relation_history` / `relation_analyze` / `relation_unfreeze` / `relation_reset`

## ⚙️ 主要配置项

### 基础开关
- `enable_injection` / `debug_mode`

### 分析模型
- `analysis_provider_id`（必填）/ `analysis_secondary_provider_id` / `analysis_timeout_seconds`

### 触发参数
- `analysis_interval_minutes` / `min_new_messages` / `enable_event_trigger`

### 冻结阈值
- `affection_freeze_threshold` / `trust_freeze_threshold`

### 存储
- `history_retention_days` / `buffer_max_size`

---

## 📝 使用说明

1. 必须配置 `analysis_provider_id`，否则分析无法执行。
2. 分析异步后台运行，不阻塞主回复。
3. 同一会话串行（asyncio.Lock 防并发），`关系分析` 可在指令触发后立即查看进度。
4. 冻结维度可通过 `解冻关系` 解冻。
5. 冷却衰减自动运行，无需干预。
6. `debug_mode` 开启后日志中会输出完整注入提示词内容。
